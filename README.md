# Git Workshop

A hands-on workshop designed to teach developers the fundamentals of version control, collaboration, and modern Git workflows.

---

## 📋 Workshop Overview

This workshop introduces participants to Git and GitHub through a combination of theory and practical hands-on labs. Participants will gain confidence using Git locally, collaborating with teams, managing branches, resolving conflicts, and recovering from common mistakes.
 
**Format:** Instructor-led with hands-on labs  
**Audience:** People new to Git or wanting to strengthen their Git skills

---

## 🎯 Learning Objectives

By the end of this workshop, participants will be able to:

- ✅ Understand core Git concepts and terminology
- ✅ Use Git locally with confidence
- ✅ Collaborate effectively using branches and GitHub
- ✅ Resolve merge conflicts
- ✅ Recover from common mistakes
- ✅ Understand and apply modern team workflows

## 🧪 Lab Structure

The workshop includes 6 progressive labs in the **Labs/** folder:

### [Lab 1: Git Basics — My First Repo](Labs/Lab-1-Git-Basics.md)
Learn the fundamental Git workflow and create your first repository.
- Initialize a repository
- Understand the staging area
- Create and view commits
- View repository history

### [Lab 2: Branching Fundamentals](Labs/Lab-2-Branching-Fundamentals.md)
Master branches for independent, safe development.
- Create and switch branches
- Understand branch isolation
- Merge feature work safely
- Visualize branch history

### [Lab 3: GitHub Collaboration Workflow](Labs/Lab-3-GitHub-Collaboration.md)
Work with remote repositories and implement team collaboration.
- Create and connect GitHub repositories
- Push code to the cloud
- Create Pull Requests
- Review and merge teammate changes

### [Lab 4: Merge Conflicts](Labs/Lab-4-Merge-Conflicts.md)
Learn to handle conflicts that arise when multiple developers edit the same code.
- Understand how merge conflicts occur
- Read Git conflict markers
- Resolve conflicts manually
- Complete merges successfully

### [Lab 5: Rebasing and Clean History](Labs/Lab-5-Rebasing-Clean-History.md)
Maintain a clean, linear commit history.
- Understand rebasing vs. merging
- Clean up feature branches
- Rewrite commit history safely

### [Lab 6: Undoing Mistakes / Git Recovery](Labs/Lab-6-Git-Recovery.md)
Learn recovery techniques for common Git problems.
- Undo commits
- Recover lost work
- Revert changes safely

## 💡 Key Concepts

### Git vs. GitHub
- **Git:** A version control system that tracks code history locally
- **GitHub:** A cloud platform for hosting and collaborating on Git repositories

### Why Use Git?
Git is not just storage — it's a **time machine + collaboration system** for files:
- 📜 **Full History:** Every change is tracked with who, what, when, and why
- ⏮️ **Easy Undo:** Revert to previous versions with one command
- 🔀 **Safe Collaboration:** Multiple people can work simultaneously without overwriting each other
- 🌳 **Branching:** Experiment safely in isolation before merging to main
- 🔒 **Backup:** Your code is stored locally and in the cloud

## 🛠 Common Git Commands Reference

| Command | Purpose |
|---------|---------|
| `git init` | Initialize a new repository |
| `git clone <url>` | Clone a remote repository |
| `git status` | View current repository status |
| `git add <file>` | Stage files for commit |
| `git commit -m "message"` | Commit staged changes |
| `git push` | Push commits to remote repository |
| `git pull` | Fetch and merge remote changes |
| `git branch` | List or create branches |
| `git switch <branch>` | Switch to a different branch |
| `git merge <branch>` | Merge a branch into current branch |
| `git log` | View commit history |

## ❓ FAQ

**Q: Do I need to be a programmer to take this workshop?**  
A: No, but basic familiarity with the command line is helpful. We'll cover everything step-by-step.

**Q: Can I use Git without GitHub?**  
A: Yes! Git works locally on your machine. GitHub is optional and is used for cloud storage and team collaboration.

**Q: What if I make a mistake?**  
A: That's expected! Git has tools to undo almost anything. Lab 6 covers recovery techniques.

**Q: Do I need to memorize all the commands?**  
A: No. Focus on understanding the concepts. You can look up commands as needed.
